Change Log¶

Stormpath is currently on Version 1 of its API, and all APIs use the /v1/ namespace. All changes to the API and documentation are therefore arranged by date, in descending order.

2016-07-01¶

  • New: Added documentation for new Custom Data Search feature. The bulk of the new information is in Reference chapter under Custom Data Search, with a smaller section 3.3.2. Searching for Accounts with Custom Data in Account Management.

2016-06-30¶

  • New: Tenants can now be assigned a Custom SMTP Server. New documentation has been added in the Account Management Chapter section Customizing Your SMTP Server, as well as a new SMTP Server section under the Tenant resource in the Reference chapter.
  • New: Directories are now able to restrict user email domains using email domain whitelists and blacklists.

2016-06-29¶

  • New: New ID Site Hosting Guide added to ID Site chapter.

2016-06-20¶

  • Modified: When Exchanging the ID Site JWT for an OAuth Token, the grant_type should now be stormpath_token.
  • Fixed: Various Resources in REST API Reference had attributes with incorrect lengths of “1 < N <= 255”. This has been corrected to be “0 < N < 256”.

2016-06-15¶

  • New: The SAML section of the Authentication chapter now contains instructions for how to configure login via ADFS.

2016-06-09¶

  • New: Account API Keys now have optional name and description attributes.
  • New: Added information about Request UUIDs to Reference chapter.
  • New: REST Errors now return the Request UUID in the error body in a new requestId attribute.
  • Modified: OAuth Access and Refresh Tokens now have a maximum TTL of 10 years (P10Y). The OAuth Policy documentation has been updated to reflect this.
  • New: Stormpath Token Type (stt) header now included with Access and Refresh Tokens, as well as ID Site Assertion and SAML Account Assertion JWTs.

2016-06-07¶

  • New: Added language switcher to left-side navigation bar. You can use it to check out our SDK Documentation. If you are on a page (for example Authorization), it will take you to the equivalent page (if applicable) in the SDK of your choosing. Currently only the PHP SDK is available in this format.

2016-06-03¶

  • New: Added screenshots to 7.5. Using ID Site for Multi-tenancy section.

2016-06-01¶

  • New: Updated ID Site chapter with two new sections: 7.5. Using ID Site for Multi-tenancy and 7.6. ID Site & Single Sign-On.

2016-05-25¶

  • Fixed: Adding missing usd claim to ID Site Authentication JWT documentation.

2016-05-18¶

  • New: Adding a new Account or Group to an Application or Organization in Account Management chapter.

2016-05-11¶

  • New: Password Policy’s Strength resource information updated with new preventReuse attribute. Also added use case information to Account Management chapter in new section How to Restrict Password Reuse.
  • New: Added missing Attribute Statement Mapping Rules, Service Provider Metadata objects to Directory’s Provider section of Reference chapter. Also added Provider Data to Account section of that chapter. Objects were previously documented and discussed in Authentication Chapter but were not included in Reference chapter.
  • New: Added additional information about attributes of Stormpath collections.
  • New: Added simplified ERD to Data Model section of About chapter.
  • Fixed: Added missing kid header to SAML Account Assertion JWT and ID Site Assertion JWT documentation.

2016-04-29¶

  • Fixed: Updated Social Login information for LinkedIn to reflect the fact that Stormpath can handle the exchange of an Authorization Code for an Access Token.

2016-04-28¶

  • New: Added information to LDAP Agent section in Reference chapter to clarify which attributes are optional, and which ones are only present for regular LDAP vs Active Directory agents. Also added clarifying note to Authentication chapter that explains that Directory, Provider, and Agent must all be passed at the same time in order to create an LDAP Directory.

2016-04-22¶

  • New: Added new SAML Authentication configuration instructions for Ping Identity.
  • New: Added information about Mapping SAML Attributes for Okta.

2016-04-21¶

  • New: Added passwordModifiedAt attribute to Account Resource table, as well as to list of Searchable Attributes. Example usage also added to new section in Account Management chapter.

2016-04-18¶

  • New: Added Stormpath Error Codes page, with list of REST Errors along with explanations.

2016-03-16¶

  • Fixed: Fixed error in ID Site Authentication JWT. organizationNameKey should be onk and showOrganizationField should be sof.

2016-03-04¶

  • New: Added simplified Entity Relationship Diagrams to Quickstart, Account Management, and Authentication sections.
  • Modified: Updated Account Management and Reference chapters to clarify that Accounts can be added to Organizations directly.

2016-03-01¶

  • New: Information about new multipart/alternative email type added to Reference section on Email Templates and Account Management section on customizing email templates via REST.

2016-02-24¶

  • New: Added section on how to route users to their tenant.

2016-02-19¶

  • New: Multi-tenancy chapter significantly expanded. Now includes more information about using Groups vs Directories to model tenants.

2016-02-16¶

  • New: Added information about IdP-initiated SAML Authentication.
  • Modified: Updated Application’s SAML Policy resource in Reference Chapter to include new defaultRelayStates endpoint.

2016-02-10¶

  • Changed Sphinx Theme to Read The Docs.

2016-02-02¶

  • New: Updated SAML Section of Authentication chapter to include IdP-specific set-up guides.
  • New: Added missing organizationAccountStoreMappings resource to Reference chapter.

2016-01-20¶

  • New: Added Change Log.
  • Modified: Directory type language changed. “Social”, “Mirror” and “SAML” Directories are all actually “Mirror Directories”, and the language in the Account Management and Authentication chapters has been updated to reflect this.
  • New: Created top-level Mirror Directories section that discusses all Mirror Directories in general and how to support multiple Mirror Directories.
  • Modified: Updated information about Master Directories and how they function.
